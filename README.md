# アクキー・アクスタ 空間ARプレビュー

対応端末では WebXR を使って床を検出し、固定の `model.glb` を空間に配置して確認できる静的アプリです。WebXR 非対応環境では、カメラ映像の上にモデルを重ねるプレビューに自動でフォールバックします。GitHub Pages のような静的ホスティングにそのまま置けます。

## できること

- ページアクセス時にカメラプレビューを自動起動
- WebXR 対応端末では床を検出して 3D モデルを配置
- 対応ブラウザではアンカー作成を試み、配置位置を安定化
- 実寸の高さ、見せる角度、回転速度を調整
- WebXR 非対応環境ではカメラ重ね表示で雰囲気確認

## ローカル起動

静的ファイルなので、任意のローカルサーバーで配信すれば動きます。

```powershell
python -m http.server 4173
```

起動後に [http://localhost:4173](http://localhost:4173) を開いてください。

## GitHub Pages で使うとき

- `index.html`
- `app.js`
- `model.glb`

この 3 ファイルをルートに置いたまま公開すれば動きます。GitHub Pages は HTTPS なので WebXR / カメラ起動条件を満たしやすいです。

## 補足

- WebXR 空間ARは対応端末と対応ブラウザが必要です
- iPhone Safari では WebXR が使えないため、通常はカメラ重ね表示フォールバックになります
- 空間ARの開始はブラウザ仕様上、ユーザー操作でボタンを押して始める必要があります
- ブラウザでカメラ許可が必要です
